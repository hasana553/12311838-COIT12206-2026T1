Week 1: 
Through GNS3, this task will give you an introduction to basic networking concepts. The primary goal of completing this task is to help you become comfortable creating a simple network project (including building the network topology) and configuring the Linux host with a static IP address. In addition to completing this exercise, I also investigated some essential software tools (such as VirtualBox and GitHub) that can assist in managing your virtual environments and keeping track of your portfolio or other documentation about your work. Conclusion:

	To summarize, GNS3 is a good tool to help you understand how to configure networks at a basic level. You can do things like create a project, add a Linux machine, give it a static IP, and connect to it using GNS3. All of this will help you learn about IP addressing and the basic functions of network devices as you practice using simulation software (i.e., GNS3).

Completing this assignment helps me feel more comfortable using GNS3 for future programming of devices. This will make it easier for me to eventually learn more about the internet security and networking fields.


Reflection:
I had my first hands-on with GNS3 during this project, creating and managing a simple network project, which was simultaneously both exciting and challenging. Learning how virtual networks work proved to be a challenge, particularly when it came to configuring nodes prior to their start time. I quickly learned the importance of carefully editing the /etc/network/interfaces file (i.e., in terms of proper format and correct values) because even one mistake or missing value may have resulted in the inability of the IP address to be applied correctly.

Through trial and error I learned how to assign static IP addresses to devices as well as verifying the IP address by using commands available in Linux (e.g., ip address show). Rather than just having theoretical knowledge, I was then able to see practical examples of how to configure IP addressing.

I also learned how to perform basic GNS3 operations, including adding nodes, annotations, and using the browser to open the console for a particular node.

I learned how important it is to complete work up front so the device can be configured (e.g., creating devices), especially since most configuration changes are made that way (this includes troubleshooting issues that may occur if the IP address does not appear due to configuration issues).

Overall, this task allowed me to develop my confidence and skills in using network simulation tools similar to GNS3 and working with device configuration files based on Linux.
